Role Call Application Development Plan

Introduction

Role Call is a desktop application designed for writers and actors to manage and organize character information. The application will enable users to store and visualize detailed character profiles, including names, backstories, psychological indices, and relationships. The goal is to provide a user-friendly tool that enhances the creative process by allowing easy access to and manipulation of character data.

The first iteration of Role Call will be a non-web-based application using Python and Flask for the backend, along with PostgreSQL for the database. The front-end will utilize Tkinter for creating the graphical user interface.

Functional Requirements

User Authentication

  • User Registration
    • Register with a username, email, and password.
    • Validation to ensure unique username and email.
    • Password strength criteria including minimum length and mixture of letters and numbers.
  • User Login
    • Log in with username/email and password.
    • Provide "Remember Me" option for persistent login.
  • Password Recovery
    • Reset password via email.
    • Send a password reset link to the registered email.
  • User Profile Management
    • Update profile information (name, email, password).

Character Management

  • Add Character
    • Create a new character with:
      • Name
      • Backstory (multi-line text)
      • Psychological Index (multi-line text or predefined categories)
      • Relationships (select other characters and define the relationship type)
  • Edit Character
    • Edit existing character details.
    • Save and update changes in the database.
  • Delete Character
    • Delete a character.
    • Confirmation dialog before deletion to prevent accidental removal.

Character Relationships

  • Define Relationships
    • Select two characters and define their relationship (e.g., parent, friend, enemy).
    • Customizable relationship types.
  • View Relationships
    • Visualize relationships using a graph or diagram.
    • Clickable nodes to view character details.

Search and Filter

  • Search Characters
    • Search for characters by name.
    • Display search results in a list format.
  • Filter Characters
    • Filter characters based on attributes:
      • Psychological traits
      • Relationship types
      • Keywords in backstories

Data Import/Export

  • Import Data
    • Import character data from CSV or JSON files.
    • Provide a template for correct formatting.
  • Export Data
    • Export character data to CSV or JSON files.
    • Select specific characters or export all data.

Notes and Attachments

  • Add Notes
    • Add notes to each character.
    • Timestamped and editable notes.
  • Attach Files
    • Attach files (e.g., images, documents) to characters.
    • Securely store files and associate them with the correct character.

Timeline Management

  • Create Timelines
    • Create timelines for each character.
    • Include significant events (e.g., birth, major life events).
  • View Timelines
    • View character timelines in chronological order.
    • Interactive timeline for adding or editing events.

Collaboration Tools

  • Share Characters
    • Share character profiles with other registered users.
    • Define permission levels (e.g., view-only, edit).
  • Real-Time Collaboration
    • Multiple users edit the same character profile simultaneously.
    • Real-time updates for all collaborators.

Version Control

  • Track Changes
    • Log every change to a character profile.
    • View change history.
  • Revert Changes
    • Revert to a previous version of a character profile.
    • Confirmation dialog before reverting changes.

Non-Functional Requirements

  • Usability: Intuitive and easy to navigate user interface with tooltips and help documentation.
  • Performance: Quick load times and handling large data without significant lag.
  • Security: Data stored securely with encryption for sensitive information. Protection against common vulnerabilities.
  • Scalability: Ability to scale for growing numbers of users and data without performance degradation.
  • Reliability: High availability with minimal downtime and maintained data integrity.
  • Compatibility: Compatible with major operating systems (Windows, macOS, Linux).
  • Maintainability: Well-documented and modular codebase for easy maintenance and updates.
  • Accessibility: Follow accessibility guidelines for use by people with disabilities.

Tech Stack

Back-End

  • Python: Core programming language.
  • Flask: Lightweight web framework for APIs and backend logic.
  • SQLAlchemy: ORM tool for Python and PostgreSQL.
  • PostgreSQL: Robust, open-source relational database system.

Front-End

  • Tkinter: Standard GUI library for Python.
  • PyQt/Kivy: Alternative GUI libraries with advanced features and modern look.

Additional Libraries

  • WTForms: Form handling and validation within Flask.
  • Flask-SocketIO: For adding real-time features if needed later.
  • Flask-JWT-Extended: For handling authentication and authorization using JSON Web Tokens (JWT).

Development and Testing Tools

  • PyTest: Testing the Python codebase.
  • PgAdmin: Management tool for PostgreSQL databases.
  • Docker: Containerizing the application for consistent development and deployment environments.

Deployment

  • PyInstaller: Bundling the Python application into standalone executables for distribution.
  • DigitalOcean/AWS: Hosting the PostgreSQL database if remote access is needed.

Development Steps

1. Set Up Development Environment

  1. Install Python: Ensure Python 3.x is installed.
  2. Set Up a Virtual Environment: ```sh python -m venv rolecall_env source rolecall_env/bin/activate # On Windows use rolecall_env\Scripts\activate ```
  3. Install Flask and SQLAlchemy: ```sh pip install Flask SQLAlchemy Flask-JWT-Extended WTForms Flask-SocketIO psycopg2 ```

2. Create Project Structure

```text rolecall/ ├── app/ │ ├── _init_.py │ ├── models.py │ ├── routes.py │ ├── forms.py │ ├── templates/ │ └── static/ ├── gui/ │ ├── main.py │ ├── views.py │ └── assets/ ├── tests/ │ ├── test_app.py │ └── test_gui.py ├── config.py ├── run.py └── requirements.txt ```

3. Implement Core Features

  • Develop the necessary database models.
  • Set up Flask app initialization and configuration.
  • Implement routes for character management.
  • Develop the Tkinter GUI for the front-end.

4. Develop Front-End GUI

  • Design the main GUI window.
  • Implement features for adding, editing, and viewing character details.
  • Develop visualization tools for character relationships.

5. Deployment and Testing

  • Use PyTest to write tests for both backend and GUI components.
  • Build the executable using PyInstaller: ```sh pip install pyinstaller pyinstaller --onefile gui/main.py ```
  • Deploy and test the application on different operating systems.

Conclusion

The Role Call application aims to streamline character management for writers and actors. This development plan provides a detailed outline of the functional and non-functional requirements, the tech stack, and the development steps needed to create a robust and user-friendly application. By following this plan, the development team can efficiently build, test, and deploy the first iteration of Role Call.
